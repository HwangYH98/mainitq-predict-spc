# MaintiQ Predict 사용자 매뉴얼

이 문서는 설치된 MaintiQ Predict를 운영자가 바로 사용하는 절차만 정리합니다. 연구 검증, benchmark, 논문 근거는 Streamlit Admin 콘솔에서 확인합니다.

## 1. 어떤 설치본을 실행할지 선택

| 설치본 | 화면 표시 | 사용 목적 |
| --- | --- | --- |
| `MaintiQ_Predict_Lite_Setup.exe` | 빠른 점검 모드 | 작은 설치본, 빠른 CSV 점검, 시연, 일상 점검 |
| `MaintiQ_Predict_Setup.exe` | 정밀 분석 모드 | XGBoost/SHAP 기반 정밀 분석, 검증용 결과 확인 |

두 모드는 계산 방식이 다르므로 같은 CSV라도 점수나 우선순위가 다를 수 있습니다. 보고서와 운영 기록에는 한 가지 모드를 정해서 일관되게 사용하세요.

## 2. 첫 실행

1. 설치 후 시작 메뉴 또는 바탕화면 바로가기에서 MaintiQ Predict를 실행합니다.
2. 로그인 화면은 없습니다.
3. API key는 처음 실행할 때 필요하지 않습니다. AI 리포트를 새로 생성할 때만 입력합니다.

## 3. CSV 예측

1. `데이터 예측` 화면으로 이동합니다.
2. 처음 사용하는 경우 `샘플 바로 사용`을 눌러 예측 흐름을 확인합니다.
3. 회사 파일을 사용할 때는 `CSV 불러오기`를 누릅니다.
4. 컬럼 확인 표에서 자동 매핑 결과를 확인합니다.
5. `예측 실행`을 누릅니다.
6. 결과는 아래 순서로 확인합니다.
   - 요약 카드
   - 상위 위험 row
   - 위험도 그래프
   - 위험 우선순위 표
   - 결과표
7. `기본 위치 저장` 또는 `결과 CSV 저장`으로 결과 파일을 저장합니다.

## 4. AI 리포트

1. `AI 리포트` 화면으로 이동합니다.
2. 템플릿을 선택합니다.
   - 운영자용
   - 관리자용
   - 정비팀용
3. 길이를 선택합니다.
   - 짧게
   - 표준
   - 상세
4. Gemini 또는 OpenAI API key를 입력합니다.
5. `연결 확인`을 누른 뒤 `새 리포트 생성`을 누릅니다.

API key는 파일에 저장하지 않습니다. 리포트 이력에는 생성 상태, 제공자, 모델, 템플릿, 오류 메시지, 미리보기만 저장됩니다.

## 5. 작업지시

1. `작업지시` 화면으로 이동합니다.
2. `정상 샘플` 또는 `고위험 샘플`로 입력값을 빠르게 채울 수 있습니다.
3. `센서 이벤트 생성`을 누릅니다.
4. `작업지시 초안 생성`을 누릅니다.
5. 작업자 결정을 선택합니다.
   - 승인
   - 검토 필요
   - 반려
6. 메모를 입력하고 `결정 저장`을 누릅니다.

이 앱은 자동 정비 명령을 실행하지 않습니다. 작업지시는 사람이 검토하고 승인하는 흐름입니다.

## 6. 자주 보는 오류와 조치

| 메시지 | 의미 | 조치 |
| --- | --- | --- |
| 필수 컬럼이 없습니다 | CSV 컬럼명이 기준 형식과 맞지 않음 | 샘플 CSV를 참고해 컬럼명을 수정하거나 매핑을 확인 |
| 숫자 형식을 확인하세요 | 숫자 컬럼에 문자 또는 비어 있는 값이 있음 | 해당 컬럼의 값을 숫자로 정리 |
| API key 형식을 인식하지 못했습니다 | Gemini/OpenAI key 형식이 아님 | 올바른 key를 다시 입력 |
| 모델 접근 실패 | 해당 key로 선택 모델 사용 불가 | 표준 모드로 바꾸거나 key 권한 확인 |
| 저장 폴더 권한 오류 | 선택한 폴더에 파일을 쓸 수 없음 | 다른 저장 위치 선택 |

## 7. 업데이트와 오류 로그

- `업데이트 확인`: GitHub Release의 최신 버전을 확인합니다. 새 버전이 있으면 Release 페이지를 열어 설치파일을 내려받도록 안내합니다.
- `오류 로그 내보내기`: 예기치 않은 오류 로그를 ZIP 파일로 묶어 저장합니다. API key와 비밀번호는 의도적으로 저장하지 않습니다.
- v1에서는 무인 자동 업데이트를 하지 않습니다. 새 설치파일을 내려받아 직접 설치합니다.

## 8. 실제 회사 데이터로 실증하려면

실제 비용 절감 또는 탐지 시간 단축을 주장하려면 단순 센서 CSV만으로는 부족합니다. 다음 세 가지가 필요합니다.

- labeled sensor CSV: 설비 ID, timestamp, 센서값, 실제 고장 여부
- maintenance history CSV: 정비 시작/종료, 조치 유형
- downtime/cost CSV: downtime, 부품비, 인건비, 기존 점검 기준

세 파일을 Admin 콘솔의 회사 데이터 실증 화면에 넣으면 가능한 주장과 불가능한 주장이 분리된 리포트를 생성할 수 있습니다.
